# EM:Product-Review - Product Requirements Review

Product and spec review combining business validation with technical feasibility assessment.

## Usage

```
Use the em:product-review skill to review [spec or requirement description]
```

## Examples

```
Use the em:product-review skill to review spec for new payment feature
Use the em:product-review skill to validate requirements for user onboarding flow
Use the em:product-review skill to assess market fit for subscription management
```

## What This Does

### Stage 1: Business Requirements (Product Manager)
- Requirements validation (problem, solution, business value)
- GAP analysis (current vs desired state)
- Acceptance criteria review (clarity, testability)
- Market fit assessment
- User story validation (INVEST criteria)
- Business impact and ROI

### Stage 2: Technical Feasibility (Architect)
- Technical feasibility and options
- Architecture impact
- Development effort estimation
- Technical risks and dependencies

## Quality Gates

- [ ] Requirements validated
- [ ] GAP analysis completed
- [ ] Acceptance criteria reviewed
- [ ] User stories validated (INVEST)
- [ ] Business impact assessed
- [ ] Technical feasibility confirmed
- [ ] Go/No-Go decision documented
